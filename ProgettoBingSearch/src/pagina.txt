<div id="content">                          <h1 class="title">Java URL example - Download the contents of a URL</h1>                    <!-- start main content -->    <div class="node node-blog node-odd ">  <div class="dd-page">                <div class="submitted">    By Alvin Alexander. Last updated: Nov 5, 2015    </div>        <div class="content"><p>Java <em>URL</em> download FAQ: How can I download the contents of a URL using Java?</p><blockquote><p>Note: It&#39;s 2013 now and this code is a little old, but hopefully it will still get you pointed in the right direction.</p></blockquote><p>In some <a href="http://java.sun.com/">Java</a> applications you&#39;ll want to download the contents of a URL across a network. For example, I&#39;ve written two applications that do this regularly.</p><p>The first Java URL download&nbsp;application is a customized Java web robot. This robot downloads the contents of certain URL&#39;s every day, and creates an HTML page of all the anchor tags it finds on those URL&#39;s. I use this Java program to get all of the headlines I want delivered to our doorsteps any time I want - with no ads and no waiting - because the robot has already done the work for us.</p><p>The second&nbsp;Java URL download&nbsp;program is a Java application I call ServerStress. This program uses Java&#39;s <a href="http://java.sun.com/j2se/1.5.0/docs/api/java/net/URL.html">URL</a> class to download the contents of a list of URL&#39;s I&#39;ve created. This program downloads the entire list from a web server as fast as it can.</p><p>The purpose of this Java application, as you might guess from it&#39;s name, is to stress-test the web server. It&#39;s a great way of throwing a mind-numbing number of client requests against a web server in a short time, and measuring the response of the server.</p><p>In this article I&#39;ll take a look at the procedure necessary to download the contents of a URL in a Java application. In a future article I&#39;ll discuss our Java <code>ServerStress</code> program, so you can see how this method is used in a real-world application.</p><h2>My Java &quot;Get URL&quot; code</h2><p>There are about five steps required to download and print the contents of a URL using Java. I say &quot;approximately&quot;, because (a) it all depends on what you consider a step, and (b) it depends on how you handle the possible exceptions that can occur.</p><p>The code in Listing 1 shows my entire JavaGetUrl.java program. When I look back at this program, I realize that the process of dealing with the possible exceptions that can occur is more time-consuming than creating the networking aspect of the code. One thing I&#39;ve learned with Java - the networking aspect of the code is pretty easy.</p><pre class="brush: java">//------------------------------------------------------------////  JavaGetUrl.java:                                          ////------------------------------------------------------------////  A Java program that demonstrates a procedure that can be  ////  used to download the contents of a specified URL.         ////------------------------------------------------------------////  Code created by Developer&#39;s Daily                         ////  <a href="http://www.DevDaily.com" title="http://www.DevDaily.com">http://www.DevDaily.com</a>                                   ////------------------------------------------------------------//import java.io.*;import java.net.*;public class JavaGetUrl {   public static void main (String[] args) {      //-----------------------------------------------------//      //  Step 1:  Start creating a few objects we&#39;ll need.      //-----------------------------------------------------//      URL u;      InputStream is = null;      DataInputStream dis;      String s;      try {         //------------------------------------------------------------//         // Step 2:  Create the URL.                                   //         //------------------------------------------------------------//         // Note: Put your real URL here, or better yet, read it as a  //         // command-line arg, or read it from a file.                  //         //------------------------------------------------------------//         u = new URL(&quot;http://200.210.220.1:8080/index.html&quot;);         //----------------------------------------------//         // Step 3:  Open an input stream from the url.  //         //----------------------------------------------//         is = u.openStream();         // throws an IOException         //-------------------------------------------------------------//         // Step 4:                                                     //         //-------------------------------------------------------------//         // Convert the InputStream to a buffered DataInputStream.      //         // Buffering the stream makes the reading faster; the          //         // readLine() method of the DataInputStream makes the reading  //         // easier.                                                     //         //-------------------------------------------------------------//         dis = new DataInputStream(new BufferedInputStream(is));         //------------------------------------------------------------//         // Step 5:                                                    //         //------------------------------------------------------------//         // Now just read each record of the input stream, and print   //         // it out.  Note that it&#39;s assumed that this problem is run   //         // from a command-line, not from an application or applet.    //         //------------------------------------------------------------//         while ((s = dis.readLine()) != null) {            System.out.println(s);         }      } catch (MalformedURLException mue) {         System.out.println(&quot;Ouch - a MalformedURLException happened.&quot;);         mue.printStackTrace();         System.exit(1);      } catch (IOException ioe) {         System.out.println(&quot;Oops- an IOException happened.&quot;);         ioe.printStackTrace();         System.exit(1);      } finally {         //---------------------------------//         // Step 6:  Close the InputStream  //         //---------------------------------//         try {            is.close();         } catch (IOException ioe) {            // just going to ignore this one         }      } // end of &#39;finally&#39; clause   }  // end of main} // end of class definition</pre><p>Listing 1 (above): The <code>JavaGetUrl.java</code> program shows how easy it is to open an input stream to a URL, and then read the contents of the URL.</p><div class="ads ads-large"><script type="text/javascript"><!--google_ad_client = "pub-5530641073697401";/* 336x280, DD, Programming */google_ad_slot = "0471404085";google_ad_width = 336;google_ad_height = 280;//--></script><script type="text/javascript"src="http://pagead2.googlesyndication.com/pagead/show_ads.js"></script></div><h2>Java &quot;Get URL&quot; code discussion</h2><p>Because this Java &quot;download URL&quot; source code is well documented, I won&#39;t add much to the description. First, I&#39;ll point out the obvious - you&#39;ll want to put your own URL in place of the &quot;http://200.210.220.1:8080/index.html&quot; in this code. This is just a TCP/IP address I use on our internal LAN during testing.</p><p>Of course, instead of hard-wiring the URL into the Java code, it would be even better to read the URL as a command-line argument, creating the URL object &quot;u&quot; with a statement like this:</p><pre>u = new URL(args[0]);</pre><p>(You&#39;ll want to do a little error-checking in your code to make sure args[0] was supplied by the user.)</p><p>Another item to point out is that I typically generated a <code>MalformedURLException</code> whenever I botched the actual URL, doing things like mis-typing &quot;http&quot; as &quot;htp&quot;, for instance. On the other hand, I generated an <code>IOException</code> whenever I properly typed the URL syntax, but mis-typed a filename.</p><h2>Java download URL code -&nbsp;Compiling and running the program</h2><p>To compile our Java URL program (after you&#39;ve downloaded it), just type this command:</p><pre>javac <code>JavaGetUrl</code>.java</pre><p>Assuming you have the JDK installed on your system, this program should compile without warnings or errors. Next, run the program like this:</p><pre>java <code>JavaGetUrl</code></pre><p>If the program runs properly, the HTML code from the URL you&#39;ve targeted will be printed to your screen. If you&#39;d like to save the output of the program (i.e., the contents of the URL) to a file, simply redirect the output of the command like this (for DOS and UNIX systems):</p><pre>java <code>JavaGetUrl</code> &gt; url.out</pre><div class="ads ads-large"><script type="text/javascript"><!--google_ad_client = "pub-5530641073697401";/* 336x280, DD, Programming */google_ad_slot = "0471404085";google_ad_width = 336;google_ad_height = 280;//--></script><script type="text/javascript"src="http://pagead2.googlesyndication.com/pagead/show_ads.js"></script></div><h2>Java download URL&nbsp;source code</h2><p>I hope you enjoyed this article.
